⚡ AI 生产力中心 · 战略本质 v2

从卖工具到卖 Token
一次彻底的身份重塑

所有问题——激励失效、公地悲剧、推广失败——都源于同一个根因。
所有解法，也都从同一个 First Principle 推导而来。

First Principle · 第一性原理
工具服务 → AI 生产力商品化（Token）
这不是产品升级，是身份转变

旧模式：AI 中心是工具供应商，业务线是被推广对象。 功劳算 AI 中心的，代价由业务线承担 → 利益天然错位 → 推广必然失败。

新模式：AI 中心是算力基础设施提供方，业务线是Token 的买方和消费方。 Token 有价，产出可量化，谁消费谁受益 → 利益天然对齐 → 制度自洽。

❌ 卖工具服务（覆盖率/使用率） → ✅ 卖 AI 生产力（Token × 效率）
一旦确立这个转变，下面所有的障碍分析和解法设计，都是这个 First Principle 的自然推论。

底层原理

关于「AI 工作渗透率」的说明
不只是研发——这是全公司所有岗位的综合指标：研发看 AI 代码参与率，产品看文档/PRD AI 辅助率，运营看内容创作频次，数据看 SQL/报告 AI 使用率，设计看 AI 生成素材占比。每个岗位有自己的子指标，汇总为统一的「AI 工作渗透率」。
为什么这个指标替换「AI 代码贡献率」
「AI 代码贡献率」天然只覆盖研发岗，占公司总人数约 30%。剩下 70% 的产品、运营、设计、市场、数据岗位被完全排除在外——这不是 AI 提效，这是研发提效。真正的「AI 生产力」是全员命题：每个人都是 Token 的消费者，每个岗位都有 Token ROI。

①
Token 数据基础设施 — 让「AI 工作渗透率」可量化
这是整个飞轮的发动机。没有这一步，后面所有节点都无法运转
🔴 关键做功点

飞轮第一步不是「推广 AI」，而是把 AI 工作量变成可测量的数字。当前大多数公司 AI 使用是「黑箱」——没人知道谁用了多少、效果怎样。你需要建一套轻量的数据采集层，覆盖研发（代码参与率）+ 非研发（AI 工具 DAU）两端，让每季度的排行榜数据是客观的、可信的、不能注水的。

🔴 你需要主动做的事
技术侧：接入 IDE/AI 工具 API，自动采集 Token 消耗 + 代码参与率 + 文档 AI 使用频次（3–6 周工程量）
规则侧：定义各岗位子指标，制定季度汇总公式
治理侧：成立数据评审委员会（AI 中心 + 效能 BP + 业务线代表），确保数据公信力
⚠️ 最容易卡死飞轮的风险
数据依赖手工填报 → 数据失真 → 排行榜没有公信力 → 整个飞轮失效。必须技术手段保证数据来自系统。
②
公开排行榜 — 让比较可见
排行榜是「社会压力」的物理载体
🔴 做功点

数据有了还不够，必须让它公开可见、有区分度。排行榜的目标不是「给优秀者颁奖」，而是让落后者看到自己的位置——「隔壁团队 L2，我们还是 L1」这句话能触发的行动，比 100 次培训更有效。

🔴 你需要主动做的事
每季度产出「快手 AI 工作渗透率全景报告」，展示：各团队综合得分 + 各岗位子指标 + 等级分布 + 本季增长最快团队。拉 CTO/VPE 一起签发，高层背书让排行榜有杀伤力。
⚠️ 风险
区分度不够 → 比较压力为零。初期要刻意拉大差距的可见性，标注「L2 已获额外 2x 积分」。
③
比较压力激活 — Leader 主动来找你
飞轮从「被推」到「主动拉」的转折点
🟢 自驱

排行榜发布后，落后团队的 Leader 会感到压力——不是 AI 中心给的，是竞争对手和老板的视线给的。这时他们会主动来找你合作，这是飞轮转起来的信号。

🟢 这时你只需要
准备好「接球」的标准流程：主动来找你的团队进入「沙盒申请」通道，2 周内给出评估、分配资源、正式启动。不要让主动来的人等超过 2 周。
④
沙盒特区 — 消除第一次失败的代价
关键做功点：让第一批试验者不怕失败
🟡 撬动一次

即使 Leader 有意愿，他还面临理性计算：「万一失败了，我需要背锅」。沙盒的作用是打破这个对称性：失败 = 免责 + 学到了什么；成功 = 功劳归团队。参与特区就变成了「对自己有利」的事。

🔴 你需要主动做的事（只需做一次）
向管理层申请：特区期间（1 季度）团队绩效豁免 AI 转型相关失败。一次性拿到高层授权——这是最难但最关键的政治动作。
⚠️ 风险
特区团队是被分配的，不是自愿的 → 走形式，数据注水。必须坚持「自愿申请制」。
⑤
L2/L3 等级认证 — 资产化 Token 能力
等级 = 团队信用凭证，用来兑换资源
🟢 自驱

沙盒试验完成后，数据达标的团队进入等级认证。等级不是荣誉牌，是可以兑换真实资源的生产力凭证：L2 获 2x 积分系数，L3 获战略级算力配额和对外输出资格。

🟢 这时你只需要
建立标准评审流程（数据自动采集 + 季度评审委员会交叉验证），保持标准的客观性。标准松了，等级失去价值；标准严了，大家会努力达到。
⑥
积分 / 算力优势 — 资源正向循环
高 ROI 团队获得更多算力，算力带来更高 ROI
🟢 自驱

积分制把「贡献→资源」的正反馈回路制度化：贡献 Skill → 积分 → 更多 Token 配额 → 更高 ROI → 更高等级 → 更多积分。这是飞轮的内燃机，一旦启动不需要外力持续加油。

🟢 你只需要定期做
每季度更新积分兑换目录，确保可兑换的东西持续有吸引力。当积分能换到「专属模型微调机会」或「跨部门专家借调」，贡献动力最强。
⑦
Skill 库沉淀 — 知识产权化
公共品困境 → 积分分红打破搭便车
🟡 初期撬动

Skill 库是飞轮的「知识积累」节点：每次有人贡献 Skill，所有人的 Token ROI 都会提升。但公共品困境决定了初期需要积分分红机制才能让人主动贡献。

🟡 你初期需要撬动一次
自己（或 AI 中心团队）亲手建前 30 个高质量 Skill，覆盖最常见场景。当库里有 50+ 个 Skill 后，社区贡献开始自驱。
⚠️ 风险
Skill 质量太低 → 用了没效果 → 失去信任。前 50 个 Skill 必须经过 AI 中心评审，宁可慢，不能滥。
⑧
Token ROI 全面提升 — 增强排行榜说服力
飞轮转速提升的核心证明
🟢 自驱

Skill 库越丰富，所有团队的 Token ROI 就越高。ROI 数据越好，排行榜的绝对分值上升，但相对差距依然存在——全面提升只会让竞争烈度上升，飞轮转速加快。

🟢 这时你只需要
定期更新排行榜基准线（全公司中位数每季度设为新的 L1 门槛），让「站在原地等于退步」的感觉持续存在。
冷启动协议 · Cold Start Protocol
飞轮不会自动启动
你需要按顺序完成这 4 步
  • S1
    建数据管道（节点①，优先级 P0）
    接入 AI 工具 API，搭建 Token 消耗 + AI 工作渗透率的自动采集体系。数据不准，后面全是废功。
    时间估算：3–6 周 · 验收标准：数据自动采集，不依赖手工填报
  • S2
    找 2–3 个自愿特区团队（节点④，P0）
    必须是 Leader 自愿，不能是被分配。同时向高层申请「特区期失败免责」授权。提前亲手建好前 30 个 Skill。
    验收标准：Leader 签了特区协议，AI 中心种子 Skill 30+
  • S3
    发布第一份排行榜（节点②，P1）
    特区跑了 6–8 周后，有了数据差距，发布第一份全公司 AI 工作渗透率报告。5–10 个团队的数据就够产生区分度。
    核心动作：拉 CTO/VPE 一起签发，增加排行榜权威性
  • S4
    等待「第一个主动来找你的团队」（转折点）
    这是飞轮启动的信号。不是你去找的，是对方主动来要合作的。比较压力已经在发挥作用——飞轮已经在转了。
    🎯 里程碑：出现第一个「非特区、主动来找你」的团队 = 飞轮启动
