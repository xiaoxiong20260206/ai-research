📥 T1 接入层 ← 触点层
📨 多端请求接入
接收 IDE/KIM/Web/本地 四端请求，统一协议
逻辑: HTTP/WebSocket 物理: KIM平台 + Nginx
📨 多端请求接入

统一接收来自四个端的请求：

  • IDE 插件 → VS Code / JetBrains 端
  • KIM 对话 → 企业 IM 消息通道
  • Web 管理台 → 浏览器前端
  • 本地端 → macOS/Windows 本地进程
协议归一化：所有端统一走 HTTP/WebSocket
⚙️ T3 计算层 ← Agent能力层
📦 OpenClaw Agent 处理
调用Skills · 代码生成 · 沙箱执行
逻辑: Agent + Skills调度 物理: K8S Pod | 4C8G
📦 OpenClaw Agent 处理
  • Agent 运行容器：每个用户独占 Pod（4C8G）
  • Skills 调度：动态加载所需技能包
  • 代码生成：AST 级别代码修改与创建
  • 沙箱执行：隔离环境运行用户代码
K8S 弹性伸缩 · 支持 GPU 推理加速
🏪 Skills Hub 技能中心
Schema管理 · 版本控制 · 分发
逻辑: 注册+审核+版本 物理: S3 + CDN
🏪 Skills Hub 技能中心
  • 技能注册：Schema 定义 + 自动校验
  • 版本管理：语义化版本 + 灰度发布
  • 审核流：安全扫描 + 人工审批
  • CDN 分发：全球加速 + 按需拉取
S3 存储 + CDN 分发 · 支持私有技能
🚪 T2 网关层 ← 服务层
🛡️ Gateway 鉴权路由
Token验证 · 消息路由 · 协议桥接
逻辑: JWT鉴权 + 路由 物理: Node.js | 18789
🛡️ Gateway 鉴权路由
  • JWT Token 身份验证（SSO 统一登录）
  • 消息路由：根据请求类型分发到对应 Agent
  • 协议桥接：WebSocket ↔ HTTP 双向转换
  • 流量管控：限流 + 降级 + 熔断
Node.js 网关 · 端口 18789
🧠 T4 推理层 ← AgentCore
🎯 智能模型路由 + 推理
按任务复杂度选模型：简单→小杯 · 复杂→大杯
逻辑: 复杂度评估+路由 物理: 万擎+Claude | MAAS
🎯 智能模型路由 + 推理
  • 复杂度评估：自动判断任务难度
  • 小杯任务 → 轻量模型（低延迟、低成本）
  • 大杯任务 → 旗舰模型（高精度推理）
  • Fallback 机制：主模型不可用自动切换备选
MAAS 统一调度 · 万擎 + Claude 双引擎
💾 T5 存储层 ← AgentCore
🗄️ KFS 存储 + 同步
持久化 · 记忆写入 · 云↔本地同步
逻辑: 流式返回+增量同步 物理: KFS卷 | /home/node/.openclaw
🗄️ KFS 存储 + 同步
  • KFS 持久卷：配置/会话/记忆中心化存储
  • 云↔本地同步：实时增量，换设备无缝衔接
  • 流式返回：大文件分块传输
  • 秒级 Failover：Pod 故障数据自动迁移
KFS 卷挂载 · /home/node/.openclaw
📊 监控 + 日志
任务统计 · 成本追踪 · 异常告警
逻辑: 指标采集+告警规则 物理: Prometheus + Grafana
📊 监控 + 日志
  • 任务统计：完成率/延迟/错误率仪表盘
  • 成本追踪：Token 消耗 + 模型调用计费
  • 异常告警：P0 故障 < 1min 触达
  • 日志聚合：全链路 Trace ID 串联
Prometheus 指标 + Grafana 可视化
